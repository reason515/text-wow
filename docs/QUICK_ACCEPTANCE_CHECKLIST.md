# 快速验收检查清单

> ⚡ **快速版本**: 这是简化版的验收清单，用于快速验证核心功能。

---

## 🎯 核心验收项（必须检查）

### 1. 角色属性显示 ✅
- [ ] 打开角色详情面板
- [ ] 检查以下属性是否有值（不为0或undefined）:
  - 物理攻击力
  - 法术攻击力
  - 物理暴击率（显示为百分比，如 "6.5%"）
  - 物理暴击伤害（显示为百分比，如 "165%"）
  - 法术暴击率
  - 法术暴击伤害
  - 闪避率（显示为百分比，如 "6.5%"）

### 2. 战斗功能 ✅
- [ ] 开始战斗
- [ ] 观察战斗日志中的伤害数值（应该合理，不应该异常）
- [ ] 检查是否有暴击提示（如 "暴击！"）
- [ ] 检查是否有闪避提示（如 "闪避！"）
- [ ] 使用技能，检查资源消耗是否正确
- [ ] 使用技能，检查伤害/治疗效果是否正确

### 3. API响应 ✅
- [ ] 打开浏览器开发者工具（F12）
- [ ] 切换到 Network 标签
- [ ] 刷新页面
- [ ] 查看 `/api/character` 请求
- [ ] 检查响应中是否包含所有计算属性
- [ ] 检查属性值类型是否正确（数字，不是字符串）

---

## ⚠️ 如果发现问题

### 属性显示为0或undefined
1. 检查浏览器控制台是否有错误
2. 检查API响应数据
3. 参考 `FRONTEND_ACCEPTANCE_GUIDE.md` 中的"常见问题排查"部分

### 伤害数值异常
1. 记录角色的攻击力和敌人的防御力
2. 手动计算预期伤害：`max(1, 攻击力 - 防御力)`
3. 对比实际伤害和预期伤害
4. 如果差异很大，可能是计算公式问题

### 暴击/闪避不显示
1. 检查战斗日志API响应
2. 检查前端日志格式化函数
3. 验证暴击率/闪避率是否合理（应该在5%-50%之间）

---

## ✅ 验收通过标准

如果以下所有项都正常，可以认为验收通过：
- ✅ 所有属性正确显示
- ✅ 战斗可以正常进行
- ✅ 伤害计算合理
- ✅ 技能可以正常使用
- ✅ API响应正常
